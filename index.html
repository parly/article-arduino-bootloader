Arduino のブートローダーとは？
  • View on GitHub

イントロダクション

このページでは、 Arduino Uno のブートローダーについて踏み込んだ解説をしています。 ブートローダーのない環境と比べることで、ブートローダーとは何をするものなのか、感じ取っていただければ幸いです。

素の AVR マイコンの動作

このセクションでは、ブートローダーが書き込まれていない、素の ATmega328P について説明します。

素のマイコンのことですから、 Arduino のブートローダーとは何も関連がないのでは？と思えてくるかもしれません。
しかし、素の環境がわかれば、その正体が徐々に見えてくるのです――

AVR マイコンの起動

AVR マイコンの電源を入れると、プログラムの実行がはじまります。
マイコンの中に “プログラムメモリー” というフラッシュメモリーがあり、そこから読み出しながら順にプログラムを実行するようになっています。

上の図は、 ATmega328P のプログラムメモリーを表したものです。 矢印の “ブートスタートアドレス” から実行が始まり、そこから順番に、図で言うと左から右へとプログラムが実行されていきます。

AVR マイコンへの書き込み

(注: 素の AVR マイコンを理解するための説明です。 Arduino では実行しないでください。 ブートローダーを壊してしまいます。)

先ほど説明したように、プログラムメモリーからプログラムを読み出す仕組みですから、そこに書き込めばよい、ということになります。

マイコン本体には、プログラムメモリーに書き込める機能が備わっています。 これを利用してプログラムを書き込むには、リセットスイッチを押しながら (*1) SPI 通信という規格に則って (*2) 信号を送受信します。 ですから書き込むときには、動作用の基板からマイコンを取り外して、 SPI 通信で AVR マイコンと通信できる IC (*3) がついた、 AVR ライターという専用基板に取り付けるのですが……

PC からプログラムを送信すると、変換用 IC が USB からの信号を受信して、 AVR マイコンが受信できる SPI に変換します。 書き込みが終了すると、プログラムメモリは以下のようになります (*4) 。

やったね ino ちゃん！実行できるよ！
しかーし、このような利用方法では、プログラムを更新するためには毎回動作用の基板から取り外して、AVR ライターに取り付ける必要がありますね。 これでは開発効率が下がって大変です……

(*1: RESET ピンに LOW 信号を入力しながら)
(*2: SPI 通信以外にも通信方法はいくつかあるが、 Arduino のように PC の USB 端子から簡単に、とはいかない)
(*3: これ自身がマイコンで実装されることもある)
(*4: ブートローダーなし、ヒューズ bootrst=0 の場合)

Arduino のブートローダー

そこで、ブートローダーの登場ですっ!!

あると何がうれしい？

毎回動作用の基板から取り外すなどの手間を省いたり、開発のサイクルを短くしたりできます。 Arduino はブートローダーのおかげで、 USB でつないだまま手軽にプログラムを変更したり、動作させたりできるのです。

ブートローダーが書き込まれた Arduino

ブートローダーが書き込まれている新品の Arduino のプログラムメモリーは、こんな感じになっています。紫色の部分が、ブートローダーです。

ATmega328P のブートローダーは、先ほどまでの普通のプログラムとは違い、末尾部分に配置されるのが一般的なようです。 ブートスタートアドレスもプログラムメモリーの先頭ではなく、ブートローダーの先頭を指しています。 プログラムの位置こそ違いますが、電源を入れるとブートローダーから実行されるところは変わらないってことですね。

スケッチ (プログラム) のアップロード

ブートローダーは、起動直後かリセット直後だけ、 AVR ライターのように振る舞います。

マイコン本体の機能を使って書き込む場合は、 SPI 通信など、USB 以外の通信方法を使わなければいけないはずでした。 それが、ブートローダーが肩代わりすることで PC の USB からの信号を直接受信してくれるようになりました。 PC からリセット信号を送り、 AVR ライターで書き込むときと同じような信号を送信すると、自分自身に書き込んでくれるってワケです。

こんなふうに。 スケッチが先頭に配置されました！

スケッチの実行

Arduino の電源を入れると、書き込んだスケッチの実行が始まります。 ブートスタートアドレスがスケッチではなく、ブートローダーの先頭を指しているのに、スケッチが実行されるのは何故でしょうか？

実はブートローダーには、指定時間以内に PC から書き換え要求が来なかった場合、スケッチへとジャンプする処理が入っています。 ですから普段は、ブートローダーがない時と同じようにスケッチが実行されます。 これによって、開発者はブートローダーの恩恵を受けながらも、普段は特に意識せずに開発ができるのです。

ブートローダー Q&A

ブートローダーがない方が良い場合ってあるの？

ブートローダーは、32 kB のプログラムメモリーの末尾の 512 バイトに格納されています。 貴重な 32 kB ですから、大きなスケッチを書き込みたい場合、ブートローダーの容量分まで使いたいことがあるかもしれません。 そんな場合には、ブートローダーを削除してしまうことができます。

ブートローダーを削除してしまったら、復活させることはできる？

AVR ライターを持っている場合、またブートローダーを書き込めます。 素の ATmega328P を安く買い、ブートローダーを自分で書き込めば、 Arduino 互換品を作ることさえできます。

ブートローダーって、自作できるの？

本家のブートローダーのソースコードが公開されていますので、改良したり、参考にして自作したりできます。
https://github.com/Optiboot/optiboot

ほとんどの場合、する必要はないと思いますが……
